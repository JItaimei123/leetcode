给定一个链表的头节点 `head` ，返回链表开始入环的第一个节点。 *如果链表无环，则返回 `null`。*

如果链表中有某个节点，可以通过连续跟踪 `next` 指针再次到达，则链表中存在环。 为了表示给定链表中的环，评测系统内部使用整数 `pos` 来表示链表尾连接到链表中的位置（**索引从 0 开始**）。如果 `pos` 是 `-1`，则在该链表中没有环。**注意：`pos` 不作为参数进行传递**，仅仅是为了标识链表的实际情况。

**不允许修改** 链表。

**示例 1：**

![circularlinkedlist.png](images/circularlinkedlist.png)

```C++
输入：head = [3,2,0,-4], pos = 1
输出：返回索引为 1 的链表节点
解释：链表中有一个环，其尾部连接到第二个节点。
```

```C++

输出：返回索引为 0 的链表节点
解释：链表中有一个环，其尾部连接到第一个节点。
```

解法1：利用双指针

```C++
class Solution {
private:
    ListNode* meetNode(ListNode* head1, ListNode* head2) {
        if (!head1 || !head2)
            return nullptr;
        ListNode *cur1 = head1, *cur2 = head2;
        int len1 = 1, len2 = 1;
        while (cur1->next) {
            cur1 = cur1->next;
            len1++;
        }
        while (cur2->next) {
            cur2 = cur2->next;
            len2++;
        }
        if (cur1 != cur2)
            return NULL;
        int gap = abs(len1 - len2);
        ListNode *longlist = head1, *shortlist = head2;

        if (len1 < len2) {
            longlist = head2;
            shortlist = head1;
        }

        while (gap--) {
            longlist = longlist->next;
        }
        while (longlist != shortlist) {
            longlist = longlist->next;
            shortlist = shortlist->next;
        }
        return shortlist;
    }

public:
    ListNode* detectCycle(ListNode* head) {
        ListNode *slow = head, *fast = head;
        while (fast && fast->next) {
            slow = slow->next;
            fast = fast->next->next;
            if (slow == fast) {
                ListNode* meet = slow;
                ListNode* newhead = meet->next;
                meet->next = NULL;
                ListNode* entry = meetNode(head, newhead);
                meet->next = newhead;
                return entry;
            }
        }
        return NULL;
    }
};
```

解法2：利用set，在set里面存节点的指针，因为每个节点的指针是唯一的，地址也是唯一的，而**节点值不唯一**，无法用来判断“是否回到同一个节点”。再建立一个指针用于遍历节点的值，将遍历到的值存入ret中，ret是 pair<iterator, bool>，这里的auto实际上是pair<set<int>::iterator, bool>，只不过这里会自动推导

### 什么 `insert` 要返回一个 `pair`？

因为 `set` 的 `insert` 需要同时告诉你两件事：

1. **元素现在在哪里**（迭代器）—— `ret.first`

2. **插入有没有成功**（bool）—— `ret.second`

### 这两个信息都很重要，所以用 `pair` 打包在一起返回给你。这就是为什么 `ret` 本身不是迭代器，而是一个包含迭代器的“复合类型”。

再检查这个值的地址在set中存不存在，若不在就将该节点所在的**地址**插入set，若在就直接返回这个节点的指针，这就是环的入口了

注意这里区分一下地址和节点的值：

| 概念 | 定义 | 代码表达 | 类比 | 是否唯一 |
| --- | --- | --- | --- | --- |
| 指针值(地址) | 指针变量本身存的内容，是一个内存地址（如 0x100） | cur 或 &node | 门牌号（如“101号”） | 每个节点的地址是唯一的 |
| 指针指向的节点的值 | 通过指针解引用后，从内存中拿到的数据（如 5） | cur->val 或 *cur | 门牌号对应的房子里装的东西 | 可能重复（多个节点值相同） |

```C++
class Solution {
public:
    ListNode *detectCycle(ListNode *head) 
    {
        set<ListNode*> s;
        ListNode* cur = head;
        while(cur)
        {   
            auto ret = s.insert(cur);
            if(!ret.second)//若没有插入成功则说明有重复的额，这时候就找到环的入口了
            return cur;
            cur = cur->next;//若可以插入则继续执行插入步骤，直到插入失败或者cur == nullptr
        }
        return nullptr;
    }
};
```
