给你一个字符串 `croakOfFrogs`，它表示不同青蛙发出的蛙鸣声（字符串 `"croak"` ）的组合。由于同一时间可以有多只青蛙呱呱作响，所以 `croakOfFrogs` 中会混合多个 `“croak”` *。*

请你返回模拟字符串中所有蛙鸣所需不同青蛙的最少数目。

要想发出蛙鸣 "croak"，青蛙必须 **依序** 输出 `‘c’, ’r’, ’o’, ’a’, ’k’` 这 5 个字母。如果没有输出全部五个字母，那么它就不会发出声音。如果字符串 `croakOfFrogs` 不是由若干有效的 "croak" 字符混合而成，请返回 `-1` 。

**示例 1：**

```C++
输入：croakOfFrogs = "croakcroak"
输出：1 
解释：一只青蛙 “呱呱” 两次
```

**示例 2：**

```C++
输入：croakOfFrogs = "crcoakroak"
输出：2 
解释：最少需要两只青蛙，“呱呱” 声用黑体标注
第一只青蛙 "crcoakroak"
第二只青蛙 "crcoakroak"
```

思路：以`“crcoakroakcroak”`为例，创建哈希表存放`“croak”`。遍历数组，完整的一次`croak`才算一次蛙叫，主要的思想是找对应，遇到c就c++，往后遍历遇到r就往前找有没有c，有c就`c--，r++`，若没有c则这次的蛙叫不成立。比如`“crroak”`遍历到第二个r的时候查找不到c那就不符合要求。然后继续遍历遇到o就去找有没有r，最后只需要看k的数量就知道需要几只青蛙叫。在`crcoakroak`阶段，可得出除了`k=2`以外其余都是`0`，此时两只青蛙都叫完了，后面还接着的`croak`便可由这两只青蛙继续叫，也就是两只青蛙可以叫三声，此时代码对应的操作是将`k-`-，然后`c++`，再以此类推找到r就`c--，r++`，直到再次达到`k = 2`，其余为0。

一旦当当前字符找不到前驱字符时直接返回-1

```C++
class Solution {
public:
    int minNumberOfFrogs(string croakOfFrogs) 
    {
        string t = "croak";
        int n = t.size();
        vector<int> hash(n);//用数组模拟哈希表，统计每个字符出现多少次
        unordered_map<char,int> index;//记录字符对应的下标，因为要找前驱
        for(int i = 0 ; i<n ; i++)
        index[t[i]] = i;  //记录每个字符对应的下标
        for(auto ch : croakOfFrogs)
        {
            if(ch == 'c')
            {
                if(hash[n-1] != 0)//检查有没有多余的蛙可以替叫
                hash[n-1]--;//若有就拿来用
                hash[0]++;//有或没有都要++
            }
            else
            {
                int i = index[ch];//对应下标
                if(hash[i - 1] == 0)//前驱字符不存在，不是完整的蛙叫
                return -1;
                else
                hash[i-1]--;
                hash[i]++;
            }
        }
        for(int i = 0;i<n-1;i++)//检查0~n-2是否都为0，否则不是合格的蛙叫
        {
            if(hash[i] != 0)
            return -1;
        }
        return hash[n-1];
    }
};
```
