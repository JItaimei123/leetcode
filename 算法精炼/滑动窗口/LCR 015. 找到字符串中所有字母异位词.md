给定两个字符串 `s` 和 ****`p`，找到 `s` ****中所有 `p` 的 **变位词** 的子串，返回这些子串的起始索引。不考虑答案输出的顺序。

**变位词** 指字母相同，但排列不同的字符串。

**示例 1：**

```C++
输入: s = "cbaebabacd", p = "abc"
输出: [0,6]
解释:
起始索引等于 0 的子串是 "cba", 它是 "abc" 的变位词。
起始索引等于 6 的子串是 "bac", 它是 "abc" 的变位词。
```


思路：

先写整体遍历构建窗口框架，再根据题目要求往中间塞，这里用滑动窗口维护p的变位词字串，用哈希表来统计每个字母出现的次数，

```C++
while(right < n)
{
  scount[s[right]]++;
  ......
  res.push_back(left);
  right++;
}
```


```C++
class Solution {
public:
    vector<int> findAnagrams(string s, string p) 
    {
        int left = 0;
        int right = 0;
        int n1 = s.size();
        int n2 = p.size();
        vector<int> res;
        if(n1 < n2)
        return res;
        int valid = 0;//用于记录有效匹配的个数，一旦valid == n2就return res
        unordered_map<char,int> scount;
        unordered_map<char,int> pcount;
        for(auto &ch : p)
        {
            pcount[ch]++;//统计p中每个字符出现的次数
        }
        while(right < n1)
        {
            scount[s[right]]++;
            if(pcount.count(s[right]) && scount[s[right]] == pcount[s[right]])
            //当前s在right这个位置上的字母，在s当前窗口中出现的次数，是不是恰好等于这个字母在p中总共需要的次数
            //若把pcount[s[right]]改成pcount[p[right]]会导致越界
            valid++;
            while(right-left+1 > n2)
            {
                if(pcount.count(s[left]) && scount[s[left]] == pcount[s[left]])
                valid--;
                scount[s[left]]--;
                left++;
            }
            if(valid == pcount.size())
            res.push_back(left);
            right++;
        }
        return res;
    }
};
```


对于

```C++
if(pcount.count(s[right]) && pcount[s[right]] == scount[s[right]])
if(pcount.count(s[left]) && scount[s[left]] == pcount[s[left]])
```


**为什么要给所有涉及 `pcount[...]` 的读取操作强行加上 `.count()` 安检？**

因为 C++ 的哈希表里，**`[ ]` 是一头"看到不认识的东西就往家里拉"的牲口**。你查一个它没有的键，它不但不报错，还会把这个键写进表里。

- **加入窗口时安检**：是为了**防止**哈希表因为多查了一个无关字符，导致**表的大小（size）无故膨胀**。

- **移出窗口时安检**：是为了**防止**哈希表因为 0 等于 0 的隐藏逻辑，导致**有效匹配数（valid）无故减少**。

**做题技巧划线：**
以后遇到这种题目，只要涉及到 **`pcount[某某]`** 的读取，**在它前面永远加上一个 `if (pcount.count(某某))`**，你的哈希表就会永远干干净净，题目也就能 100% 通过了！
