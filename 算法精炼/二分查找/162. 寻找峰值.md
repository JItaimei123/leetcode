# [162. 寻找峰值](https://leetcode.cn/problems/find-peak-element/)

峰值元素是指其值严格大于左右相邻值的元素。

给你一个整数数组 `nums`，找到峰值元素并返回其索引。数组可能包含多个峰值，在这种情况下，返回 **任何一个峰值** 所在位置即可。

你可以假设 `nums[-1] = nums[n] = -∞` 。

你必须实现时间复杂度为 `O(log n)` 的算法来解决此问题。

对于所有有效的 `i` 都有 `nums[i] != nums[i + 1]`

## 示例

**示例 1：**

```C++
输入：nums = [1,2,3,1]
输出：2
解释：3 是峰值元素，你的函数应该返回其索引 2。
```

**示例 2：**

```C++
输入：nums = [1,2,1,3,5,6,4]
输出：1 或 5 
解释：你的函数可以返回索引 1，其峰值元素为 2；
     或者返回索引 5， 其峰值元素为 6。
```

## 题解（二分查找 · 二段性思想）

### 思路

分为三种情况：

1. 从一开始就向下，并且单调，这种情况起始点就是峰值，因为 `nums[-1]` 可以看作 `-∞`。
2. 从一开始就向上，并且单调，这种情况末尾点就是峰值，因为 `nums[n]` 同样可以看作 `-∞`。
3. 变化趋势不单调，那么可以找到这个开始变化的 **"折点"**，这个点就是峰值。

这三种情况都被二分查找涵盖其中，在左右边界查找的时候终止条件是 `left = right`，恰好一二种情况在查找过程中都会出现——在模板总结左边界查找那里的图片就能看出，当数组中的数全都小于 target 的时候，会一直往右找，直到 `left = right = nums.size()-1`；全都大于 target 就会往左找，直到 `left = right = 0`。

这个题用左右边界查找的风格都可以解答，只要与题目逻辑自洽就行，主要是考察二分思想，考察 **二段性** 的思想。

### 左中点版本

```C++
class Solution {
public:
    int findPeakElement(vector<int>& nums) 
    {
        int left = 0,right = nums.size()-1;
        while(left < right)
        {
            int mid = left+(right-left)/2;//左中点
            if(nums[mid] > nums[mid+1])//去左边找
            right = mid;//对应左中点，避免死循环
            else//去右边找
            left = mid + 1;
        }
        return left;
    }
};
```

### 右中点版本

```C++
class Solution {
public:
    int findPeakElement(vector<int>& nums) 
    {
        int left = 0,right = nums.size()-1;
        while(left < right)
        {
            int mid = left+(right-left+1)/2;//右中点
            if(nums[mid] > nums[mid-1])//去右边找
            left = mid;//对应右中点，避免死循环
            else//去左边找
            right = mid - 1;
        }
        return left;
    }
};
```
