# [724. 寻找数组的中心下标](https://leetcode.cn/problems/find-pivot-index/)

给你一个整数数组 `nums` ，请计算数组的 **中心下标** 。

数组 **中心下标** 是数组的一个下标，其左侧所有元素相加的和等于右侧所有元素相加的和。

如果中心下标位于数组最左端，那么左侧数之和视为 `0` ，因为在下标的左侧不存在元素。这一点对于中心下标位于数组最右端同样适用。

如果数组有多个中心下标，应该返回 **最靠近左边** 的那一个。如果数组不存在中心下标，返回 `-1` 。

## 示例

**示例 1：**

```C++
输入：nums = [1, 7, 3, 6, 5, 6]
输出：3
解释：
中心下标是 3 。
左侧数之和 sum = nums[0] + nums[1] + nums[2] = 1 + 7 + 3 = 11 ，
右侧数之和 sum = nums[4] + nums[5] = 5 + 6 = 11 ，二者相等。
```

## 题解

思路：假设中心下标的位置是i，构建两个前缀和数组分别为`f[i]` `g[i]`

![724.png](./images/724.png)

如图所示，这里下标是从0开始，要求两个前缀和数组的和

`f[i]`表示`[0,i-1]`区间内元素的和，`f[i] = f[i-1] + nums[i-1]`填表时`从左往右`

同理`g[i]`表示`[i+1,n-1]`区间内的元素和，`g[i] = g[i+1] + nums[i+1]`填表时`从右往左`

再比较g[i]和f[i]是否相等

### 注意处理细节：`f(0) = 0,g(n-1) = 0`

### 为什么 `f[0] = 0`？

`f[i]` 的定义是 `[0, i-1]` 区间内元素的和。

当 `i = 0` 时，区间是 `[0, -1]`，**没有元素**，所以和为 `0`。

`f[0] = 0` 是前缀和数组的**哨兵值**，方便递推计算。

---

### 为什么 `g[n-1] = 0`？

`g[i]` 的定义是 `[i+1, n-1]` 区间内元素的和。

当 `i = n-1` 时，区间是 `[n, n-1]`，**没有元素**，所以和为 `0`。

`g[n-1] = 0` 是后缀和数组的**哨兵值**，方便递推计算。

```C++
class Solution {
public:
    int pivotIndex(vector<int>& nums) 
    {
        //预处理前缀和数组
        int n = nums.size();
        vector<int> f(n,0),g(n,0);
        for(int i = 1;i<n;i++)
        f[i] = f[i-1] + nums[i-1];
        for(int i = n-2;i>=0;i--)
        g[i] = g[i+1] + nums[i+1];
        //使用前缀和数组
        for(int i = 0;i<n;i++)
        if(f[i] == g[i])
        return i;
        return -1;
    }
};
```
